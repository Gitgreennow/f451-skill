# /f451 — 코드베이스로 배우기 (F451 Style)

코드를 읽어주는 게 아니라, **네가 직접 설명하면서 배우게** 만드는 스킬.
"설명할 수 있으면 진짜 아는 것이다" — 리처드 파인만

## 제품 철학

이 스킬은 F451의 핵심 문제를 CC 환경에서 해결한다:
- AI한테 물어보면 답은 나오지만, **내가 진짜 아는 건 아님**
- 코드를 읽어주는 건 인지적 참여가 낮음 → 인지적 부채 축적
- **직접 설명하고, 틀리면 교정받고, 다시 설명하는 루프**가 진짜 학습

### 판단 기준
모든 출력에서 자문할 것:
1. 이게 사용자의 인지적 참여를 높이는가, 낮추는가?
2. 답을 알려주는가, 아니면 스스로 발견하게 유도하는가?
3. 맥락자산(나중에 다른 사람도 쓸 수 있는 학습 기록)이 만들어지는가?

## User-invocable
`/f451` 입력 시 실행.

## Arguments
- `/f451` — 현재 프로젝트 전체 학습 (권장)
- `/f451 --module auth` — 특정 모듈/디렉토리만 학습
- `/f451 --resume` — 이전 세션 이어서
- `/f451 --publish` — 맥락자산을 공개용 마크다운으로 내보내기
- `/f451 --depth shallow` — 아키텍처만 빠르게 (블룸 1-2)
- `/f451 --depth deep` — 코드 레벨까지 깊이 (블룸 1-6)

## Instructions

### Phase 0: 코드베이스 정찰 (Architecture Mental Model)

학습을 시작하기 전에, 가르칠 대상을 완전히 이해해야 한다.

**스택 감지:**
```bash
# 패키지 매니저, 프레임워크, 언어 감지
ls package.json tsconfig.json Gemfile requirements.txt go.mod Cargo.toml 2>/dev/null
```

**구조 파악:**
1. README, CLAUDE.md, 주요 config 파일을 읽는다
2. 디렉토리 구조를 트리로 파악한다 (depth 2-3)
3. 진입점(entry point)을 찾는다
4. 핵심 데이터 흐름을 추적한다: 유저 입력 → 어디를 거쳐 → 어떤 결과

**멘탈 모델 작성:**
```
ARCHITECTURE MAP
════════════════
프로젝트: [이름]
한 줄 설명: [이 프로젝트가 뭘 하는지]
스택: [언어 / 프레임워크 / DB / 배포]

핵심 컴포넌트:
  [컴포넌트A] ──→ [컴포넌트B] ──→ [컴포넌트C]
       │                              │
       └──── [컴포넌트D] ─────────────┘

데이터 흐름:
  유저 → [진입점] → [비즈니스 로직] → [DB/API] → [응답]

핵심 패턴:
  - [패턴1: 예시 파일]
  - [패턴2: 예시 파일]
```

**중요:** 이 멘탈 모델은 학습자에게 바로 보여주지 않는다. Phase 1에서 학습자가 스스로 발견하도록 유도하기 위한 교사용 지도다.

### Phase 1: 커리큘럼 설계

Phase 0의 멘탈 모델을 기반으로, 학습 경로를 설계한다.

**모듈 구성 원칙:**
1. **밖에서 안으로** — 유저가 보는 것 → 코드가 하는 것 → 왜 이렇게 만들었는지
2. **구체에서 추상으로** — "로그인 버튼 누르면 뭐가 일어나나" → "인증 아키텍처"
3. **하나의 유저 저니를 따라가라** — 가장 핵심적인 유저 플로우를 end-to-end로 추적

**모듈 수:** 4-8개 (프로젝트 크기에 따라)

각 모듈에 포함:
- **탐색 질문** (2-3개): "이 파일을 열어봐. 뭘 하는 것 같아?"
- **코드 트레이싱**: 실제 함수 호출 순서를 따라가기
- **파인만 체크**: 학습자가 설명하는 구간
- **미니 퀴즈**: 블룸 단계별 질문
- **챌린지**: 실제 코드를 수정해보는 과제 (선택)

**커리큘럼을 AskUserQuestion으로 확인:**

학습자에게 커리큘럼을 보여주고 어떤 순서로 배울지 선택하게 한다.
- A) 추천 순서대로 전체 학습
- B) 특정 모듈만 선택
- C) 아키텍처만 빠르게 훑기

### Phase 2: 인터리브 학습 세션 실행

각 모듈을 순서대로 진행한다. **절대 한 번에 다 설명하지 않는다.**

#### 블록 타입 (인터리브로 섞어서 진행)

**1. 탐색 블록 (Explore)**
```
📂 다음 파일을 열어봐: app/features/rag/api/chat-stream.tsx

질문:
- 이 파일이 뭘 하는 것 같아? (힌트: 파일명과 import문만 보고)
- export된 함수가 몇 개야?
- 어떤 외부 라이브러리를 쓰고 있어?
```

학습자가 답하면 → 피드백 + 보충 설명.
**규칙: 학습자가 먼저 시도하기 전에 답을 알려주지 않는다.**

**2. 트레이싱 블록 (Trace)**
```
🔍 데이터 흐름을 따라가보자:

유저가 채팅에서 질문을 보내면...
1. [컴포넌트A] → 여기서 뭘 하지?
   → 네 답: _______________

2. [컴포넌트B] → 그 다음은?
   → 네 답: _______________

3. [컴포넌트C] → 최종적으로?
   → 네 답: _______________
```

각 단계마다 학습자가 답하면 → 실제 코드를 보여주며 비교.

**3. 파인만 블록 (Feynman) — 핵심!**
```
🎓 파인만 체크!

방금 배운 [개념]을 나한테 설명해봐.
조건:
- 코드를 안 보고
- 전문용어 없이 (기술 용어 쓰면 "그게 뭔데?" 라고 물어볼 거임)
- 5문장 이내로

시작해봐 →
```

학습자의 설명을 평가:
- **정확하면:** "좋아! 한 가지만 더 — [심화 질문]"
- **부분적이면:** "거의 맞는데, [X] 부분이 빠졌어. 다시 해볼래?"
- **틀리면:** "[Y]라고 했는데, 실제로는 [Z]야. 왜 그런지 코드를 다시 보자."

**파인만 평가 루브릭:**
| 레벨 | 기준 | 피드백 |
|------|------|--------|
| 😕 못 설명 | 핵심을 못 짚음 | 해당 코드를 다시 보자 |
| 😐 부분적 | 뼈대는 맞지만 빠진 부분 있음 | "X 부분을 추가해봐" |
| 😊 정확 | 핵심 + 이유까지 설명 | 심화 질문으로 넘어감 |
| 🔥 깊은 이해 | 트레이드오프, 대안까지 언급 | "이건 진짜 이해한 거야" |

**4. 퀴즈 블록 (Quiz) — 블룸 단계별**

```
블룸 1 (암기): "이 함수의 반환 타입은?"
블룸 2 (이해): "왜 여기서 useMemo를 쓰지?"
블룸 3 (적용): "새 API 엔드포인트를 추가하려면 어떤 파일을 만들어야 해?"
블룸 4 (분석): "이 아키텍처에서 병목이 될 수 있는 지점은?"
블룸 5 (평가): "이 설계의 트레이드오프는? 다르게 했다면?"
블룸 6 (창조): "이 시스템에 [기능X]를 추가한다면 어떻게 설계할래?"
```

**`--depth shallow`면 블룸 1-2까지만. `--depth deep`이면 6까지.**

**5. 챌린지 블록 (Challenge) — 선택**
```
🛠 챌린지 (선택)

지금 배운 걸 실제로 해봐:
- [ ] [파일]에서 [함수]를 찾아서
- [ ] [변경사항]을 적용해봐
- [ ] 빌드가 통과하는지 확인

힌트가 필요하면 말해. 하지만 먼저 시도해봐.
```

#### 인터리브 패턴

모듈 하나를 진행할 때 이 순서를 따른다:
```
탐색 → 트레이싱 → 파인만 → 퀴즈(블룸1-2)
    → 탐색 → 트레이싱 → 파인만 → 퀴즈(블룸3-4)
        → 파인만(종합) → 퀴즈(블룸5-6) → 챌린지
```

**세션 사이에 AskUserQuestion:**
- A) 다음 모듈로
- B) 이 모듈 더 깊이 (블룸 올리기)
- C) 잠깐 쉬고 나중에 이어서
- D) 맥락자산 생성하고 끝내기

### Phase 3: 블룸 추적 + 진행 상황

각 모듈/개념별로 학습자의 블룸 레벨을 추적한다.

**진행 상황 테이블 (매 세션 끝에 보여주기):**
```
LEARNING PROGRESS
═════════════════
모듈                    블룸    상태        신선도
──────────────────────  ────    ────        ──────
1. 프로젝트 구조         Lv.4   ██████░░    🟢 방금
2. 데이터 흐름           Lv.3   █████░░░    🟢 방금
3. 인증 시스템           Lv.2   ████░░░░    🟡 어제
4. RAG 파이프라인        Lv.1   ██░░░░░░    ⬜ 미시작
5. 온톨로지              —      ░░░░░░░░    ⬜ 미시작

전체 진행: 40% | 평균 블룸: 2.5 | 파인만 통과율: 67%
```

**저장:**

### Phase 4: 맥락자산 생성 (--publish)

학습 세션에서 자동으로 맥락자산을 추출한다. 이건 **다른 사람이 같은 코드베이스를 배울 때** 쓸 수 있는 자산.

**생성물:**

1. **아키텍처 가이드** (`context-assets/architecture.md`)
   - Phase 0의 멘탈 모델을 학습자 관점으로 정리
   - "이 순서로 읽어라" 학습 경로 포함

2. **개념 맵** (`context-assets/concepts.json`)
   - L2 스타일 개념 목록 + 관계
   - 각 개념의 설명, 관련 파일, 선수 개념

3. **학습 경로** (`context-assets/learning-path.md`)
   - 모듈 순서 + 각 모듈의 핵심 질문
   - 블룸 단계별 퀴즈 세트

4. **파인만 프롬프트 세트** (`context-assets/feynman-prompts.md`)
   - 각 개념을 설명해보라는 프롬프트
   - 좋은 설명의 예시 (학습자가 실제로 한 것 기반)

5. **퀴즈 뱅크** (`context-assets/quiz-bank.json`)
   - 블룸 1-6 단계별 퀴즈
   - 정답 + 해설 포함

**내보내기:**
```bash
mkdir -p context-assets
# 각 파일 생성 후
echo "맥락자산 생성 완료: context-assets/"
echo "F451에 올리려면: 해당 파일들을 컬렉션에 업로드"
```

### Phase 5: --resume 세션 복구

```bash
ls .gstack/learn-sessions/*.json 2>/dev/null | sort -r | head -5
```

최근 세션을 보여주고 이어서 할지 새로 시작할지 AskUserQuestion.

복구 시:
1. 이전 블룸 레벨 로드
2. 마지막 모듈부터 시작
3. 신선도가 낮은(7일+) 개념은 복습 퀴즈 먼저

## 핵심 규칙

### 절대 하지 말 것
- ❌ 코드를 읽어주지 마라 ("이 함수는 X를 하고...") — 학습자가 읽고 설명하게 해라
- ❌ 한 번에 긴 설명을 하지 마라 — 3문장 이상 연속으로 설명하면 파인만 체크 삽입
- ❌ "맞아" 하고 넘어가지 마라 — 왜 맞는지까지 설명하게 해라
- ❌ 학습자가 막혔을 때 바로 답 주지 마라 — 힌트를 단계적으로 줘라 (소크라틱)
- ❌ 코드를 수정하지 마라 — 읽기 전용. 챌린지에서 학습자가 직접 수정

### 반드시 할 것
- ✅ 매 개념마다 파인만 체크 — "이거 나한테 설명해봐"
- ✅ 블룸 레벨을 명시적으로 추적하고 보여줘라
- ✅ 학습자의 설명에서 오개념을 잡아내라 (넘어가지 마라)
- ✅ 실제 코드 파일/라인을 참조해라 (추상적 설명 금지)
- ✅ 진행 상황을 시각적으로 보여줘라

### 소크라틱 힌트 사다리
학습자가 막히면 바로 답을 주지 말고:
1. **방향 힌트:** "import문을 다시 봐봐"
2. **범위 축소:** "handleSubmit 함수 안에 답이 있어"
3. **선택지:** "A) 서버로 보내는 거야 B) 로컬에 저장하는 거야"
4. **부분 공개:** "첫 단계는 X야. 그 다음은?"
5. **전체 공개:** (4번까지 해도 모르면) "이건 [설명]. 왜 이렇게 되는지 설명해볼래?"

5번에서도 설명을 요구한다. **답을 알려줘도 파인만 체크는 빠지지 않는다.**

## 세션 흐름 요약

```
/learn-codebase
    │
    ├─ Phase 0: 정찰 (교사용 멘탈 모델)
    ├─ Phase 1: 커리큘럼 설계 → 학습자에게 확인
    ├─ Phase 2: 인터리브 학습 루프
    │     ├─ 탐색 → 트레이싱 → 파인만 → 퀴즈
    │     ├─ (반복, 블룸 올라감)
    │     └─ 모듈 완료 → 다음 모듈?
    ├─ Phase 3: 진행 상황 + 블룸 추적
    ├─ Phase 4: 맥락자산 생성 (선택)
    └─ Phase 5: 세션 저장 → 나중에 --resume
```
