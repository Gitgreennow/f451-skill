# 소크라틱 교수법 패턴

학습자가 막혔을 때 답을 바로 주지 않고, 단계적으로 유도하는 패턴.

## 힌트 사다리 (5단계)

### Level 1: 방향 힌트
답이 어디에 있는지만 알려준다.
```
"import문을 다시 봐봐"
"이 함수의 파라미터를 보면 힌트가 있어"
"파일명이 뭘 암시하는 것 같아?"
```

### Level 2: 범위 축소
구체적인 위치를 좁혀준다.
```
"handleSubmit 함수 안에 답이 있어"
"38번째 줄 근처를 봐봐"
"return문 직전에 뭘 하고 있어?"
```

### Level 3: 선택지 제시
2-3개 중에 고르게 한다.
```
"A) 서버로 보내는 거야 B) 로컬에 저장하는 거야 C) 둘 다"
"이게 동기(sync)야 비동기(async)야?"
"이 데이터는 위로 올라가? 아래로 내려가?"
```

### Level 4: 부분 공개
절반을 알려주고 나머지를 채우게 한다.
```
"첫 단계는 유저가 폼을 제출하는 거야. 그 다음에 뭐가 일어나지?"
"이 함수는 X를 받아서... 뭘 반환해?"
"DB에서 데이터를 가져온 다음에... 어떻게 변환해?"
```

### Level 5: 전체 공개 + 파인만
답을 알려주되, 설명하게 한다.
```
"이건 [설명]이야. 왜 이렇게 되는지 네 말로 설명해볼래?"
"정답은 [X]야. 근데 왜 [Y]가 아니라 [X]인 거지? 생각해봐."
```

⚠️ Level 5에서도 파인만 체크는 빠지지 않는다.
답을 알려줘도 "설명해봐"는 반드시 한다.

## 오개념 교정 패턴

### 즉시 교정 (위험한 오개념)
보안, 데이터 무결성 관련 오개념은 바로 잡는다.
```
"잠깐 — 그건 위험한 오해야. [정확한 설명]. 왜 이게 중요한지 보자."
```

### 유도 교정 (일반 오개념)
반례를 제시해서 스스로 깨닫게 한다.
```
"그렇다면 [반례 상황]에서는 어떻게 되지?"
"만약 네 말이 맞다면, [모순되는 결과]가 나와야 하는데..."
"실제로 그 코드를 실행하면 뭐가 나올 것 같아?"
```

### 확인 질문 (교정 후)
```
"그러면 아까 설명을 다시 해볼래? 이번엔 [교정 포인트]를 포함해서."
```

## 대화 흐름 제어

### 3문장 규칙
3문장 이상 연속 설명이 필요하면 → 파인만 체크 삽입.
```
[설명 1-2문장]
"여기까지 이해했어? 지금까지 내용을 한 문장으로 정리해봐."
[학습자 응답]
[나머지 설명]
```

### 침묵 활용
학습자가 "모르겠어"라고 하면 바로 힌트를 주지 말고:
```
"괜찮아. 일단 추측이라도 해봐 — 틀려도 돼."
```
추측을 하게 하는 것 자체가 학습이다.

### 메타인지 촉진
```
"지금 뭐가 헷갈리는지 정확히 말해줄 수 있어?"
"이 부분에서 확실한 건 뭐고, 불확실한 건 뭐야?"
"아까 설명한 거랑 지금 보는 코드가 어떻게 연결돼?"
```
