# 블룸 택소노미 — 질문 설계 가이드

학습 블록에서 퀴즈/파인만 질문을 만들 때 이 레퍼런스를 참조한다.

## 6단계 + 질문 패턴

### Lv.1 암기 (Remember)
**학습자가 할 수 있는 것:** 사실을 떠올린다
**질문 패턴:**
- "이 함수의 반환 타입은?"
- "이 파일은 어디에 있어?"
- "이 테이블의 PK는?"
- "[용어]가 뭐야?"

### Lv.2 이해 (Understand)
**학습자가 할 수 있는 것:** 의미를 설명한다
**질문 패턴:**
- "이 함수가 뭘 하는지 네 말로 설명해봐"
- "왜 여기서 [패턴X]를 쓰지?"
- "이 코드를 비개발자한테 어떻게 설명할래?"
- "[A]와 [B]의 차이는?"

### Lv.3 적용 (Apply)
**학습자가 할 수 있는 것:** 배운 걸 새 상황에 쓴다
**질문 패턴:**
- "새 API 엔드포인트를 추가하려면 어떤 파일을 만들어야 해?"
- "이 패턴을 [새 상황]에 적용하면?"
- "[기능X]를 구현하려면 어디부터 시작해?"
- "이 버그를 고치려면 어떤 파일을 봐야 해?"

### Lv.4 분석 (Analyze)
**학습자가 할 수 있는 것:** 구조를 분해하고 관계를 파악한다
**질문 패턴:**
- "이 시스템에서 병목이 될 수 있는 지점은?"
- "[모듈A]와 [모듈B]는 어떻게 연결돼 있어?"
- "이 코드의 의존성 그래프를 그려봐"
- "데이터가 [A]에서 [B]까지 가는 경로를 추적해봐"

### Lv.5 평가 (Evaluate)
**학습자가 할 수 있는 것:** 판단하고 비교한다
**질문 패턴:**
- "이 설계의 트레이드오프는?"
- "다른 방법으로 했다면 뭐가 달라졌을까?"
- "[접근법A] vs [접근법B] — 어떤 게 더 낫고 왜?"
- "이 코드의 가장 큰 약점은?"

### Lv.6 창조 (Create)
**학습자가 할 수 있는 것:** 새로운 것을 설계한다
**질문 패턴:**
- "이 시스템에 [기능X]를 추가한다면 어떻게 설계할래?"
- "이 아키텍처를 처음부터 다시 만든다면?"
- "[문제Y]를 해결하는 새 모듈을 설계해봐"
- "이 패턴을 개선한 버전을 제안해봐"

## 파인만 평가 기준

| 점수 | 기준 | 학습자가 보여줘야 하는 것 |
|------|------|-------------------------|
| 1/4 😕 | 못 설명 | 핵심을 못 짚거나 순환 논리 |
| 2/4 😐 | 부분적 | 뼈대는 맞지만 중요한 부분 누락 |
| 3/4 😊 | 정확 | 핵심 + 이유 + 적절한 예시 |
| 4/4 🔥 | 깊은 이해 | 트레이드오프, 대안, 한계까지 |

## 블룸 레벨 업 조건

| 현재 → 다음 | 필요한 증거 |
|-------------|-----------|
| 0 → 1 | 파일을 열고 탐색 완료 |
| 1 → 2 | 파인만 2/4 이상 + 퀴즈 Lv.2 정답 |
| 2 → 3 | 퀴즈 Lv.3 60%+ 정답 |
| 3 → 4 | 파인만 3/4 이상 + 퀴즈 Lv.4 정답 |
| 4 → 5 | 트레이드오프 분석 + 대안 제시 |
| 5 → 6 | 새 설계 제안 + 근거 설명 |
